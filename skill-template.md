# Overview

このSkillは、◯◯な入力（例: 要件定義書、議事録、ソースコードなど）を受け取り、
△△な出力（例: テスト項目案、アクションアイテム、レビューコメントなど）を生成することを目的としています。

- 他のSkillやエージェントとの関係があれば、ここに概要だけを書いても構いません。

# Inputs

このSkillは、以下の情報を入力として想定します。

- 必須入力
  - 例: 「要件定義書の本文（プレーンテキスト）」
  - 例: 「会議の議事録（タイムスタンプ付きでも可）」

- 任意入力
  - 例: 「対象システムの概要や前提条件」
  - 例: 「優先度付けの基準（ビジネス優先 / リスク優先 など）」

入力として受け取るテキストの形式や、あれば望ましい構造（例: セクション見出し、箇条書き）もここに明記してください。

# Outputs

このSkillは、以下の形式で出力を行います。

- 出力内容
  - 例: テスト項目案の一覧
  - 例: アクションアイテムの箇条書き
  - 例: コードレビューコメントのリスト

- 出力フォーマット
  - できるだけ一定の構造になるよう、Markdownまたは擬似JSONでフォーマットを指定してください。
  - 例（Markdown形式）:

    ```markdown
    # タイトル

    ## セクション1タイトル

    - 箇条書きの項目1
    - 箇条書きの項目2

    ## セクション2タイトル

    - ...
    ```

  - 例（JSONライクな形式）:

    ```json
    {
      "items": [
        {
          "id": "ITEM-001",
          "title": "項目タイトル",
          "detail": "詳細な説明",
          "note": "補足があれば記載"
        }
      ]
    }
    ```

必要であれば、必ず含めるべきフィールドや、空にしてよいフィールドもここで明示してください。

> **出力形式は別ファイルに分割することが望ましい**

# Steps

このSkillを実行する際は、次のステップに従ってください。

1. 入力テキストを全体として読み、内容の概要や目的を把握する。
2. 必要であれば、前提条件や制約事項を抽出して整理する。
3. Skillの目的（例: テスト項目案の抽出、アクションアイテムの抽出など）に沿って、
   入力から関連する情報を見つけ出し、候補を洗い出す。
4. 洗い出した候補を整理し、優先度・カテゴリ・ラベルなどが必要であれば付与する。
5. 「Outputs」セクションで指定したフォーマットに従って、出力を構造化してまとめる。
6. 最後に、重複・明らかな誤り・目的から外れた項目がないかを簡単に見直し、必要に応じて修正する。

上記は汎用的な流れなので、具体的なSkillに合わせてステップを足したり削ったりしてください。

# Guidelines

このSkillを用いて出力を生成する際は、次のガイドラインを守ってください。

- 文体・トーン
  - 文体: 「です・ます」調で統一する（必要に応じて変更してください）。
  - 専門用語は必要な範囲で使用し、初出時に簡単な説明を加えることを推奨します。

- 内容の粒度
  - 出力は、実務でそのまま使えるレベルの具体性を目指してください。
  - 抽象的すぎる表現（例: 「テストする」「確認する」だけ）は避け、
    「何を」「どのように」「どの観点で」行うかが分かるように書いてください。

- 禁止事項 / 注意事項
  - 想像や推測だけで事実を断定しないでください。
  - 入力に存在しない情報を、あたかも存在するかのように書かないでください。
  - セキュリティやプライバシーに関わる内容が含まれる場合は、
    不必要な個人情報や機密情報を出力に含めないように注意してください。

- その他のルール
  - 出力は常に「Outputs」セクションで指定したフォーマットに従ってください。
  - 指定されたフォーマットで表現しにくい情報がある場合は、
    「note」や「補足」などのフィールドを追加して対応してください。
