# Session Override

## 概要

この Skill は、`sessions/` の最新セッション記録を読み込み、前回の作業内容を把握して今回の作業方針を立てる。

引き継いだ記録をもとに、現在のブランチ、GitHub Issues、作業ツリーの状態を確認し、次に取るべき行動を整理する。

## 実行条件

- `sessions/` に過去のセッション記録が存在すること。
- Git リポジトリの状態を確認できること。
- GitHub Issues を確認する場合は、対象リポジトリが分かっていること。

## 入力

- 必須入力:
  - `sessions/YYYY-MM-DD-session-handover.md` の最新版
- 任意入力:
  - 確認したい GitHub リポジトリ
  - 今回再開したい作業テーマ

## 実行手順

1. この Skill の `SKILL.md` を読む。
2. `sessions/` の最新日付ファイルを読む。
3. 前回の完了事項、未完了事項、注意点を整理する。
4. Open な GitHub Issues を確認する。
5. 現在のブランチと作業ツリーを確認する。
6. 今回の作業方針を提案する。

## 出力

以下を報告する。

- 前回セッションの要約
- 現在のブランチと作業ツリー状態
- Open な GitHub Issues の有無
- 今回の作業候補
- 注意すべき未完了事項

## 検証観点

- 最新の session handover を読んでいること。
- Git 状態を実際に確認していること。
- GitHub Issues を確認した場合は、確認対象と結果を明示していること。
- 前回記録にない内容を事実として補完していないこと。

## 注意

- `sessions/` の記録は接続先 Fess や作業環境に依存するローカル情報として扱う。
- 推測と記録上の事実を分ける。
- 作業方針は提案に留め、破壊的操作はユーザー確認なしに実行しない。
