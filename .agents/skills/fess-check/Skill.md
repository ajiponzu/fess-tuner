# Overview

このSkillは、Fessサーバー（`FESS_BASE_URL`）に対してヘルスチェックリクエストを送信し、
サーバーが正常に稼働しているかどうかを確認して結果を報告することを目的としています。

- Dockerコンテナ上で動作するFessサーバーの状態確認に使用します。
- fess-search や fess-crawl を実行する前の事前確認としても利用できます。
- エンドポイント: `GET /api/v1/health`（認証不要・公開API）
- 正常時のレスポンス: `{"data": {"status": "green", "timed_out": false}}`

# Inputs

このSkillは、以下の情報を入力として想定します。

- 必須入力
  - なし（`.env` の設定で実行可能）

- 任意入力
  - なし

# Outputs

このSkillは、以下の形式で出力を行います。

- 出力内容
  - Fessサーバーの稼働状態（正常 / 異常）
  - HTTPステータスコード
  - エラーが発生した場合の詳細情報

- 出力フォーマット

  ```markdown
  # Fess サーバー ヘルスチェック結果

  - ステータス: 正常（200 OK） / 異常
  - サーバーURL: [FESS_BASE_URL]
  - メッセージ: [詳細メッセージ]
  ```

# Steps

このSkillを実行する際は、次のステップに従ってください。

1. 以下のコマンドを実行してヘルスチェックを行う。

   ```
   python .Codex/skills/fess-check/main.py
   ```

2. コマンドの出力を確認する。
3. HTTPステータスコード 200 かつ `data.status` が `"green"` であればサーバーが稼働中と報告する。
4. リクエストが失敗した場合、エラーの詳細を報告し、Dockerコンテナが起動しているか確認を促す。

# Guidelines

このSkillを用いて出力を生成する際は、次のガイドラインを守ってください。

- 文体・トーン
  - 簡潔かつ明確に結果を伝えてください。
  - 異常時は「何が問題か」「何を確認すべきか」を具体的に示してください。

- 内容の粒度
  - ステータスコードや具体的なエラーメッセージは必ず含めてください。
  - 推測による原因断定は避け、確認すべき事項を提案してください。

- 禁止事項 / 注意事項
  - 実際のコマンド出力に存在しない情報を付け加えないでください。
