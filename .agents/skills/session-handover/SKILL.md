# Session Handover

## 概要

この Skill は、セッション終了時に今回の作業内容を整理し、次回の作業へ引き継ぐための Markdown を作成する。

会話の要約だけではなく、セッション中の行動、変更ファイル、判断事項、必要に応じた Git / GitHub の状態を含め、実際の進捗に基づいた記録を残す。

## 実行条件

- セッション中の主要な作業内容を把握していること。
- 必要に応じて Git 状態を確認できること。
- `sessions/` ディレクトリが存在すること。
- 出力内容に秘密情報が含まれていないこと。

## 入力

- 必須入力:
  - セッションの会話内容
  - 実際に作成、更新、削除した成果物
- 任意入力:
  - Git リポジトリの状態
  - 変更されたファイル
  - 差分の有無
  - 直近のコミット
  - 現在のブランチ
  - 関連する GitHub Issue / Pull Request / 進捗メモ

## 実行手順

1. この Skill の `SKILL.md` を読む。
2. 必要に応じて `template.md` を読む。
3. セッション全体の対話内容と実施内容を振り返る。
4. 今回扱ったファイル、成果物、判断事項を洗い出す。
5. 可能であれば Git と GitHub の状態を確認する。
6. 完了したこと、未完了のこと、保留中のことを分類する。
7. Web 検索を使用した場合は、何について使用したかと取得情報の日付を記録する。
8. 次回に最初にやるべき作業を、実行可能な粒度で列挙する。
9. `sessions/YYYY-MM-DD-session-handover.md` として出力する。
10. 出力内容を読み直し、不足や誤りがあれば訂正する。

## 出力

- 出力先: プロジェクト直下の `sessions/`
- 出力形式: `template.md` に従う Markdown
- ファイル名形式: `sessions/YYYY-MM-DD-session-handover.md`

## 検証観点

- 完了事項、未完了事項、保留事項が分かれていること。
- 実際に確認した Git / GitHub 状態だけを書いていること。
- 次回作業が実行可能な粒度で書かれていること。
- 重要なリスクや注意点が省略されていないこと。
- 秘密情報が含まれていないこと。

## 注意

- 会話だけを根拠に進捗度を断定しない。
- 実際には確認していない Git や GitHub 状態を確認済みのように書かない。
- 次回タスクを抽象的な表現だけで済ませない。
- 重要な未完了事項やリスクを省略しない。
- 読み返しにくい長文だけで埋めない。
