# Session Handover

## Overview

この skill は，セッション終了時に今回の作業内容を整理し，次回の作業へ引き継ぐための markdown を作成するためのものです．  
出力先は `sessions` ディレクトリとし，人間がレビュー・編集・再配置しやすい実務向けの引き継ぎメモを作成します．

この skill は，会話の要約だけを行うものではありません．  
セッション中の対話や行動に加えて，必要に応じて GitHub や git リポジトリの状態も確認し，実際の進捗に基づいた記録を残します．

## Inputs

- セッションの会話
- git リポジトリの状態
- 変更されたファイル
- 差分の有無
- 直近のコミット
- 現在のブランチ
- 関連する GitHub 上の Issue，Pull Request，進捗メモ
- 実際に作成，更新，削除した成果物

## Outputs

- 出力先は プロジェクト直下の`sessions` ディレクトリとする
- 出力形式は `template.md` に従う
- ファイル名の形式は下記である
  - `sessions/2026-05-05-session-handover.md`

## Steps

1. セッション全体の対話内容と実施内容を振り返る
2. 今回扱ったファイル，成果物，判断事項を洗い出す
3. 可能であれば git と GitHub の状態を確認する
4. 完了したこと，未完了のこと，保留中のことを分類する
5. `WebSearch`を使用した場合，何について使用したか記載し，取得情報の日付も記録する
6. 次回に最初にやるべき作業を，実行可能な粒度で列挙する
7. `sessions` ディレクトリに markdown ファイルとして出力する
8. 出力内容を読み直し，不足や誤りがあれば訂正する

## Guidelines

- 会話だけを根拠に進捗度を断定しない
- 実際には確認していない Git や GitHub 状態を，確認済みのように書かない
- 次回タスクを抽象的な表現だけで済ませない
- 重要な未完了事項やリスクを省略しない
- 読み返しにくい長文だけで埋めない
- 句読点には「，」, 「．」を使用する
